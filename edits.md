# Pitch Sequence Prediction: MLB → Polish Ekstraliga Transfer Learning

**MSDS Capstone Project** | Quoc Dat Nguyen | BUID U55798231

## Overview

This repository implements a data-driven pitch-sequence model that:

1. **Trains on MLB data** (millions of richly-featured pitch records)
2. **Adapts to Polish Ekstraliga** (∼7,000 PAs, pitch results only) via transfer learning + feature masking
3. **Powers an in-game decision-support tool** to recommend optimal pinch-hitter or relief-pitcher substitutions

## Model Pipeline

### Phase 1 — Baseline: Markov Chain

Two Markov models trained entirely on Ekstraliga data:

| Model | State | Predicts |
|---|---|---|
| `OutcomeMarkov` | (balls, strikes) at PA end | Outcome category (K, BB, HIT…) |
| `PitchTransitionMarkov` | (last token, balls, strikes) | Next pitch token |

Strength: interpretable, fast. Weakness: no memory beyond current count.

### Phase 2 — LSTM with Tendency Features

Architecture:
```
Pitch token IDs → Embedding (32d) → LSTM (128d, 2 layers)
                                          ↓
Tendency features → MLP (15d → 32d)       ↓
                              → Concat → Classifier → 9 outcome classes
```

**Transfer learning steps:**
1. Pre-train on MLB data (all features, including physical)
2. Freeze LSTM encoder weights
3. Zero-mask MLB-only physical features (velocity, spin rate, etc.)
4. Fine-tune tendency MLP + classifier head on Ekstraliga

### Phase 3 — Decision Support

The `DecisionEngine` wraps the trained model to answer:
- *"Given bases loaded, 1 out in the 8th, which of our available pinch hitters maximises expected run value?"*
- *"Which relief pitcher best suppresses this upcoming lineup segment?"*

Ranking is done by **Leverage-weighted Expected Run Value (ERV)**:
```
ERV = Σ P(outcome) × ΔRunValue(outcome) × LeverageIndex
```

## Training Arguments

```
python src/train.py [OPTIONS]

  --data     Path to Ekstraliga xlsx (required)
  --model    markov | lstm | both  (default: both)
  --epochs   Number of LSTM training epochs (default: 30)
  --batch    Batch size (default: 64)
  --lr       Learning rate (default: 1e-3)
  --hidden   LSTM hidden dim (default: 128)
  --embed    Token embedding dim (default: 32)
  --layers   LSTM layers (default: 2)
  --dropout  Dropout rate (default: 0.3)
  --outdir   Output directory (default: runs/<timestamp>)
```

## MLB Data Collection

```bash
# Pull 500 random games from 2024 regular season
python src/mlb_api.py --season 2024 --n_games 500 --outdir data/mlb/

# Pull June–August 2024
python src/mlb_api.py --start 2024-06-01 --end 2024-08-31 --outdir data/mlb/
```

MLB Stats API is free and requires no API key. The parser maps MLB pitch codes
and event names to the same token vocabulary used by the Ekstraliga pipeline,
allowing the same `features.py` / `lstm_model.py` code to handle both datasets.

## Key Design Decisions

**Feature masking for transfer learning**  
MLB records include pitch velocity, spin rate, and plate coordinates.
Ekstraliga only has pitch results and counts. Rather than training two
separate models, we train one model on MLB with a "masking" flag that
zeros out physical features — so the model *can* use them if available
but learns to also predict well without them.

**Chronological train/val/test split**  
We sort by date and split 80/10/10. This prevents future game statistics
from leaking into the model's training data, which would overstate performance.

**Class-weighted loss**  
PA outcomes are imbalanced (singles most common, triples rare). We use
inverse-frequency class weights in the cross-entropy loss so the model
doesn't collapse to predicting "1B" for every at-bat.
